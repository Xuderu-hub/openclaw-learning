CHAPTER 01Gateway — 交通枢纽

🏢 Gateway 在架构中的位置

Gateway是OpenClaw的入口层，所有外部消息都经过它。它不做任何"思考"，只做三件事：接收 → 路由 → 转发。

类比：酒店前台 — 接电话、转接、挂电话。它不处理业务，只做分发。

📥 1. 接收消息

接收来自各个渠道的消息：QQ消息、微信消息、网页聊天、Telegram等。不解析内容，只管"收"。

🛤️ 2. 路由分发

根据消息携带的Session ID，找到对应的Agent和上下文，把消息送过去处理。

📤 3. 结果返回

Agent处理完毕后，Gateway把生成的回复送回对应的聊天通道（QQ/微信/网页等）。

⏰ Session 管理

  • 每个聊天会话 = 一个独立Session
  • Session记录：对话历史、用户信息、当前上下文
  • 不同渠道的Session完全隔离，互不干扰
  • Session通过Session ID唯一标识

⏰ Cron 调度

  • Gateway内置Cron调度器
  • 到了预设时间，自动触发任务（不依赖用户消息）
  • 触发后生成一个内部事件，交给Agent执行
  • 你的每日热点推送就是这样自动跑的

CHAPTER 02Session — 对话房间

🚪 什么是Session？

Session = 对话房间。每个和你对话的渠道，都有一个独立的Session，拥有独立的上下文和记忆。

💬
QQ
Session-A
💬
微信
Session-B
🌐
网页
Session-C
📱
Telegram
Session-D

🔒 上下文隔离

  • 你在QQ问的事情，不会影响网页的对话内容
  • 每个Session有自己独立的"记忆"
  • 切换渠道时，上下文不会丢失也不会串台
  • Session销毁后，对话历史也随之清除

🔑 Session ID 的作用

  • 每条消息都携带Session ID
  • Gateway根据ID找到对应的Session
  • Session里保存了：对话历史、用户信息、配置参数
  • ID是唯一的，格式类似 session-xxx-xxx

🗂️ Session 存储什么？

对话历史

你和AI之间的所有消息记录

用户信息

用户名、ID、渠道标识

运行时状态

当前模型、工具配置等

CHAPTER 03Agent — 理解与决策

🤖 Agent 是 OpenClaw 的"大脑"

Gateway只管转发，Agent才是真正"思考"的地方。它理解你的意思，决定做什么，选择合适的工具，然后生成回复。

⚙️ Agent 的四大职责

🔍 意图理解

理解你真正想问什么，而不只是字面。同一句话在不同上下文里意思不同。

⚙️ 工具选择

决定是否调用工具、调哪个工具。Agent知道每个Skill能做什么。

📚 上下文管理

从Session中读取历史消息，向Memory请求相关记忆，保持对话连贯性。

💬 回复生成

综合所有信息（工具结果+上下文+记忆），用自然语言生成最终回复。

🔄 Agent 处理消息的完整流程

💬
收到消息
→
🧠
理解语义
在上下文中理解意图
→
🔌
决策
要不要调用工具？
→
🔧
调用Skill
执行工具获取结果
→
💬
生成回复
综合结果组织语言

整个过程在毫秒级完成，用户感知不到延迟

📖 上下文（Context）

  • 当前Session的对话历史（最近N轮消息）
  • System Prompt（AI的角色设定）
  • Skill返回的结果
  • Memory返回的相关记忆

🎯 Tool/Skill 选择机制

  • 每个Skill都有描述（SKILL.md里定义）
  • Agent根据任务需求匹配合适的Skill
  • 必要时串联多个Skill完成复杂任务
  • Skill结果再交回Agent组织最终回复

CHAPTER 04Memory — 记忆系统

🧠 为什么需要Memory？

Agent的上下文（Context）只能记住"最近几轮对话"。Memory系统让AI能够记住更早的、重要的事情，跨越会话积累知识。

🏗️ 记忆分层架构

Working Memory

当前会话的上下文，随对话结束而消失

大小：有限（几千到几万个token）

Episodic Memory — 事件记忆

memory/YYYY-MM-DD.md 每日原始记录

记录每天发生的raw事实，由Agent每日自动写入

Short-term Memory — 短期记忆

memory/.dreams/ Dreaming暂存区

凌晨3点Dreaming运行时，临时存放待整合的记忆碎片

Long-term Memory — 长期记忆

MEMORY.md 结构化精华

萃取后的结构化知识：用户偏好、重大决策、经验教训

Procedural Memory — 程序记忆

memory/procedures.md 工作流模式

记录常见的操作步骤和最佳实践

🌙 Dreaming — 自动记忆整合

⏰ 何时运行？

每天凌晨 3:00 Asia/Shanghai，自动触发

用户睡觉时，AI也在工作

🔄 三阶段流程

  • Light 轻量整理
  • REM 深度整合
  • Deep 精华萃取

📊 六维评分

  • 相关性 0.30
  • 频率 0.24
  • 查询多样性 0.15
  • 时效性 0.15
  • 整合度 0.10
  • 概念丰富度 0.06

🔬 Dreaming 的工作原理

  • 读取 memory/ 下的每日笔记
  • 读取 MEMORY.md 现有长期记忆
  • 分析哪些记忆值得保留、哪些可以合并
  • Rehydration（再水化）确保不固化过时内容
  • 评分低的记忆被淘汰或归档
  • 新的重要信息写入MEMORY.md

CHAPTER 06Cron — 定时任务

⏱️ 什么是Cron？

Cron是Gateway内置的定时任务调度器。你可以把它理解为"闹钟"——到了预设的时间，自动触发一段代码执行，不需要你手动操作。

⚙️ Cron 核心配置项

📅 schedule

何时执行。使用cron表达式定义时间规则。

  • 0 9 * * * = 每天9:00
  • 0 3 * * * = 每天3:00
  • */30 * * * * = 每30分钟

📦 payload

执行什么内容。

  • systemEvent 发送系统事件
  • agentTurn 启动Agent执行任务

🎯 sessionTarget

在哪里执行。

  • main 主会话
  • isolated 独立会话（推荐）
  • current 当前会话

📢 delivery

如何返回结果。

  • announce 推送通知给用户
  • webhook POST到指定URL
  • none 不推送

🔄 定时任务完整流程

⏰
时间到达
cron表达式匹配
→
🏧
Gateway
Cron调度器触发
→
🤖
Agent
执行payload内容
→
📤
delivery
推送结果给用户

📰 每日热点推送

每天 9:00 自动抓取抖音、微博热搜，整理后推送给用户。

🌙 Dreaming

每天 3:00 自动运行记忆整合系统，优化长期记忆。

🔧 自定义任务

用户可添加任意定时任务：备份文件、检查服务、发送提醒等。

CHAPTER 07Tools — 工具系统

🔧 Tools vs Skills

Tools是OpenClaw内置的基础能力接口（文件操作、网络请求等）。Skills是封装好的完整技能包（调用Tools、执行业务逻辑）。Skills底层依赖Tools工作。

🛠️ 常用Tools一览

📁 文件操作

  • read — 读取文件
  • write — 写入文件
  • edit — 编辑文件

💻 命令执行

  • exec — 执行Shell命令
  • process — 管理运行中的进程

🌐 网络

  • web_fetch — 获取网页内容
  • web_search — DuckDuckGo搜索
  • MiniMax__web_search — MiniMax搜索

💬 消息

  • sessions_send — 跨会话发消息
  • sessions_spawn — 启动子会话
  • cron — 定时任务管理

🖼️ 媒体生成

  • image_generate — 生成图片
  • video_generate — 生成视频
  • music_generate — 生成音乐

🤖 AI能力

  • MiniMax__understand_image — 图片理解
  • image — 视觉分析
